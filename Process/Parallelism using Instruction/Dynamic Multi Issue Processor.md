### 동적 다중 내보내기 프로세서
-----
1. 동적 다중 내보내기 프로세서는 슈퍼스칼라(Superscalar) 프로세서 혹은 단순히 슈퍼스칼라라고 함
   - 슈퍼스칼라(Superscalar) : 프로세서가 실행 중 명령어를 선택하여 한 클럭 사이클에 2개 이상 명령어를 실행할 수 있도록 해주는 고급 파이프라이닝 기법
   - 제일 간단한 슈퍼스칼라 프로세서는 명령어를 순차대로(In-Order) 내보내고 주어진 클럭 사이클에 몇 개의 명령어를 내보낼지 결정
   - 좋은 성능을 얻기 위해서는 컴파일러가 명령어를 스케줄하여 종속성 있는 명령어들의 위치를 멀리 떨어뜨리고, 명령어 내보내기율을 증가시켜야 함
   - 컴파일러 스케줄링을 사용하기는 하지만, 단순한 수퍼스칼라 프로세서와 VLIW 프로세서는 결정적인 차이가 존재
     + VLIW에서는 컴파일러가 잘 스케줄해야 프로그램이 제대로 실행되지만, 슈퍼스칼라는 스케줄링 여부와 상관없이 하드웨어가 코드의 올바른 실행을 보장
     + 뿐만 아니라 컴파일된 코드는 프로세서의 내보내기율이나 파이프라인 구조와 상관없이 항상 올바르게 동작
     + VLIW 프로세스는 그렇지 않으므로, 다른 프로세서 모델로 코드를 옮길 때 다시 컴파일해야 함
     + 다른 정적 내보내기 프로세서의 경우 구현이 달라져도 코드가 올바르게 실행되기는 하지만, 성능이 너무 저하되므로 실제로는 재컴파일이 필요

  2. 많은 슈퍼스칼라 프로세서가 동적 내보내기를 결정하는 기본 틀을 확장하여 동적 파이프라인 스케줄링(Dynamic Pipeline Scheduling)을 포함하고 있음
     - 동적 파이프라인 스케줄링 : 지연을 피할 수 있도록 명령어 실행 순서를 바꾸는 재정렬을 하드웨어가 지원하는 것
     - 동적 파이프라인 스케줄링은 해저드와 지연을 최대한 피하면서 주어진 클럭 사이클에 실행할 명령어들을 선택
     - 데이터 해저드를 피하는 간단한 예
