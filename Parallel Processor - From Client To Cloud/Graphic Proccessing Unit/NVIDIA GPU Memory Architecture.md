### NVIDIA GPU 메모리 구조
-----
1. NVIDIA GPU 메모리 구조
<div align="center">
<img src="https://github.com/user-attachments/assets/a0407f6d-22a2-4b69-93e4-1f727183b808">
</div>

  - 멀티스레드 SIMD 프로세서 내 존재하는 온칩 메모리를 지역 메모리(Local Memory)
  - 멀티스레드 SIMD 프로세서 내 있는 SIMD 레인은 지역 메모리를 서로 공유하지만 다른 멀티스레드 SIMD 프로세서와는 공유하지 않음
  - 전체 GPU와 모든 스레드 블록이 공유하는 DRAM은 칩 외부에 존재하며 GPU Memory(GPU 메모리)라고 부름

2. 응용 프로그램의 모든 워킹셋(Working Set)을 담을 수 있는 큰 캐시 대신에 GPU는 전통적으로 작은 크기의 스트리밍 캐시를 사용하고, DRAM의 긴 지연시간을 감추기 위해 SIMD 명령어 스레드들의 멀티스레딩을 대대적으로 사용
   - 그 이유는 워킹셋의 크기가 수백 MB가 되어, 멀티코어 마이크로프로세서의 마지막 단계 캐시에 들어가지 않음
   - DRAM 메모리 지연시간을 감추기 위해 하드웨어 멀티스레딩을 사용하므로, 시스템 프로세서에서 캐시에 사용되던 칩 영역을 컴퓨팅 자원과 많은 레지스터 구현에 사용
   - SIMD 명령어의 많은 스레드의 상태를 저장하기 위해 레지스터가 많이 필요

3. 메모리 지연시간을 감추자는 것이 GPU의 기본 철학이지만, 최신 GPU와 벡터 프로세서는 캐시를 추가하고 있음
   - 이 캐시는 전통적 캐시라기보다 GPU 메모리에 대한 접근 요구를 줄여주는 대역폭 필터(Bandwidth Filter)나 멀티스레딩으로 지연을 감출 수 없는 소수의 변수를 위한 가속기로 볼 수 있음
   - 함수를 호출할 때는 지연이 중요하므로 스택 프레임이나 함수 호출, 레지스터 스필(Register Spill)을 위한 지역 메모리가 캐시와 잘 어울림
   - 온칩 캐시에 접근하면 칩 외부의 DRAM을 여러 번 접근하는 것보다 훨씬 에너지 소비가 작으므로 캐시는 에너지도 절약

4. 상위 수준에서 GPU는 SIMD 명령어를 확장한 멀티코어 컴퓨터와 유사한 면이 많음
   - 유사점과 차이점
<div align="center">
<img src="https://github.com/user-attachments/assets/5fe78dd9-aca5-4587-99f9-8eaf2c7026f5">
</div>

  - GPU가 더 많은 프로세서와 훨씬 더 많은 수의 SIMD 레인을 가지고 있지만, 둘 다 여러 개의 SIMD 레인을 갖는 프로세서로 구성된 MIMD
  - GPU가 훨씬 더 많은 스레드를 지원하지만 둘 다 프로세서의 효율을 높이기 위해 하드웨어 멀티스레딩을 사용
  - 두 가지 모두 캐시를 상요하는데, GPU는 작은 크기의 스트리밍 캐시를 사용하는 반면, 멀티코어 컴퓨터는 되도록 워킹셋 전체를 담을 수 있을만큼 큰 다단계 캐시를 사용
  - GPU의 실제 메인 메모리가 훨씬 작기는 하지만, 둘 다 64비트 주소 공간을 사용하며, GPU는 페이지 계층에서 메모리 보호를 지원하지만 아직 요구 페이징(Demand Paging)은 지원하지 않음

5. SIMD 프로세서도 벡터 프로세서와 비슷
   - GPU의 SIMD 프로세서들은 독립적인 MIMD 코어처럼 동작하는데 이것은 벡터 컴퓨터들이 여러 벡터 프로세서를 가지고 있는 것과 같음
   - 가장 큰 차인점은 멀티스레딩으로, GPU에서는 기본적인 기능이지만 대부분 벡터 프로세서에는 없음

6. 컴퓨터 구조의 계보를 따라 올라가도 CPU와 GPU의 공통 조상을 찾을 수 없음
   - 두 혈통이 다르므로 GPU는 컴퓨터 구조 커뮤니티에서 사용하는 용어를 사용하지 않음
   - 이로 인해 GPU가 어떤 것이고 어떻게 동작을 하는지 이해하는데 혼란이 발생
   - 이런 혼란을 줄이기 위해 NVIDIA GPU에서 공식적으로 사용하는 언어에 대한 그림
