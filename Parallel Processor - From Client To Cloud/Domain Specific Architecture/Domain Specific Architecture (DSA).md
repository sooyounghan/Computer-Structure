### 도메인에 특화된 구조
-----
1. 범용 컴퓨터와 달리 응용 도메인 하나에 맞추어 만들어진 특수 목적용 컴퓨터로, 협소한 범위의 작업만 수행하지만 작업을 수행하는 성능은 매우 뛰어남
2. 컴퓨터의 새로운 표준은 운영체제와 같은 통상적인 큰 프로그램을 수행하는 표준적인 프로세서와 도메인에 특화된 프로세서로 구성하는 것
3. DSA의 다섯 가지 원리
   - 데이터 이송 거리를 최소화하기 위해 전용 메모리 사용
     + 범용 마이크로프로세서의 다단계 캐시는 프로그램의 데이터를 최적으로 전송하기 위해 많은 면적과 에너지를 사용
     + 예를 들어 2-way 집합 연관 캐시는 소프트웨어로 제어되는 같은 크기는 스크래치패드 메모리보다 2.5배나 더 많은 에너지를 사용
     + DSA에 정의에 의하면 DSA를 위한 컴파일러 작성자와 프로그래머는 도메인을 잘 이해하고 있으므로 하드웨어적 방법으로 데이터를 이동하려고 애쓸 필요가 없음
     + 대신 소프트웨어로 제어하는 메모리를 사용하는 데이터 이동을 줄이며, 이 메모리는 도메인의 특정 함수들에 맞추어진 전용 메모리임

   - 고급 마이크로구조 최적화 기법을 포기함으로써 절약되는 자원을 더 많은 연산 유닛과 더 큰 메모리에 사용
     + CPU와 GPU는 비순차 실행, 추정, 멀티스레딩, 멀티프로세싱, 선인출, 다단계 캐시등과 같은 자원을 많이 사용하는 최적화에 사용
     + 하지만 도메인이 협소한 DSA는 프로그램 실행 특성을 잘 이해할 수 있으므로 이런 자원을 많은 처리 장치나 큰 온칩 메모리에 더 잘 활용할 수 있음

   - 도메인에 적합한 가장 쉬운 형태의 병렬성을 활용
     + DSA의 대상 도메인은 거의 항상 내재된 병렬성을 가지고 있음
     + DSA가 결정해야 할 핵심은 어떻게 이 병렬성을 이용할 것인가와 어떻게 이 병렬성을 소프트웨어가 볼 수 있게 할 것인지임
     + 목표는 도메인 병렬성의 자연스러운 크기를 기준으로 DSA를 설계하고 프로그래밍 모델에 병렬성을 보이게 하는 것
     + 예를 들어 SIMD 방식으로 데이터 수준 병렬성을 활용하는 것이 도메인에 적합하다면 MIMD보다 SIMD가 프로그래머나 컴파일러 작성자에게 더 쉬울 것

   - 도메인에 필요한 최소 크기로 데이터 크기와 데이터 형을 줄임
     + 많은 도메인의 응용에서 메모리가 병목이므로, 더 작은 데이터형을 이용하면 실효 메모리 대역폭을 증가시키고, 온칩 메모리 활용도를 높일 수 있음
     + 작고 단순한 데이터를 사용한다면 동일한 칩 면적과 에너지 예산으로 더 많은 산술 유닛을 집적시킬 수 있음

   - 도메인에 특화된 프로그래밍 언어를 사용해 DSA에 코드를 이식
