### 클러스터, 창고 규모 컴퓨터와 기타 메세지 전달 멀티프로세서
-----
1. 주소 공간을 공유하는 대신 프로세서들이 자신만의 실제 주소 공간을 갖도록 하는 방법이 존재
2. 다수의 전용 주소 공간(Private Address Space)을 사용하는 멀티프로세서의 전형적인 구성

   - 이 방식의 멀티프로세서는 명시적인 메세지 전달(Message Passing, 명시적으로 정보를 보내고 받음으로써 프로세서 간의 통신을 수행하는 방식) 방식으로 프로세서 간 통신이 이루어지므로 메세지 전달 컴퓨터라고 부름
   - 시스템이 메세지 전송 루틴(Send Message Routine, 전용 메모리 컴퓨터에서 한 프로세서가 다른 프로세서로 메세지를 전송하는 루틴)과 메세지 수신 루틴(Receive Message Routine, 전용 메모리 컴퓨터에서 한 프로세서가 다른 프로세서로부터 메세지를 수신하는 루틴)을 가지고 있는 경우, 송신 프로세서는 메세지 전송 시점을 알고 수신 프로세서는 메세지 도착 시점을 알기 때문에 프로세서 간 조정이 메세지 전달 자체로 이루어짐
     + 송신측이 메세지 도착 여부를 확인할 필요가 없을 때는 수신 프로세서가 송신 프로세서에게 확인 메세지를 보낼 수 있음
   - 근거리 네트워크(LAN)을 사용하여 구성한 클러스터보다 통신 성능의 절대적 측면에서 우수

3. 캐시 일관적 공유 메모리 컴퓨터보다 메세지 전달을 이용해 통신하는 컴퓨터를 만드는 것이 훨씬 쉬움
   - 또한 프로그래머 입장에서는 통신이 명시적으로 표현되기 때문에 공유 메모리 컴퓨터의 암묵적 통신과 달리 성능을 어느 정도 예측할 수 있다는 장점이 존재
   - 반면에 단점은 순차적인 프로그램을 메세 지전달 컴퓨터에 이식시키는 것이 힘듬
   - 통신이 일어나야 할 부분을 사전에 모두 표시하지 않으면, 프로그램이 동작하지 않을 것임
   - 캐시 일관적 공유 메모리에서는 하드웨어가 통신이 필요한 데이터를 파악하기 때문에 이식이 쉬움
   - 멀티코어 마이크로프로세서는 공유 메모리를 사용하는 반면, 클러스터 노드들은 메세지 전달을 이용해 서로 통신

4. 클러스터(Cluster, 메세지 전달 멀티프로세서를 구성하기 위해 표준 네트워크 스위치를 통해 입출력으로 연결된 컴퓨터들의 집합)가 오늘날 가장 널리 쓰임
   - 클러스터의 각 노드는 독자적 메모리와 운영체제를 가지고 있음
   - 반면에 마이크로프로세서 내 코어들은 하나의 운영체제를 사용하면서 칩 내 고성능 연결망으로 연결되어 있으며, 멀티칩 공유 메모리 시스템은 메모리 연결망으로 연결되어 있음 : 메모리 연결망은 대역폭이 더 크고 지연시간은 작으므로, 공유 메모리 멀티프로세서의 통신 성능이 훨씬 좋음
   - 클러스터는 근거리 네트워크로 연결된 독립적인 컴퓨터로 구성되므로 시스템 전체를 중단시키지 않고 컴퓨터 하나를 대체하는 것이 공유 메모리 멀티프로세서 경우에 비해 훨씬 쉬움
   - 클러스터의 경우 서버가 하나 고장나면 부드럽게 성능을 떨어뜨릴 수 있으므로 신용도가 좋아짐
   - 클러스터는 완전한 컴퓨터와 독립적이면서도 확장 가능한 네트워크로 구성되므로 클러스터 위에서 돌아가는 응용들을 중단시키지 않고 시스템을 확장하는 것도 쉬움
