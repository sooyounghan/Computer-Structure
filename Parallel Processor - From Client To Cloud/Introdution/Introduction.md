-----
### 서론
-----
1. 멀티프로세서(Multi-Processor)
   - 최소한 2개 이상의 프로세서들로 구성된 컴퓨터 시스템
   - 1개의 프로세서로 구성된 컴퓨터 시스템을 일컫는 단일 프로세서에 대비되는 용어

2. 멀티프로세서는 확장 가능한(Scalable) 성능을 제공할 뿐 아니라 전력 효율도 개선할 수 있음
3. 멀티프로세서 소프트웨어에 확장성이 있으면 하드웨어가 일부 고장나더라도 동작이 중지되지 않도록 설계 가능
   - 즉, 프로세서 n개 중에서 1개가 고장이 나면 남은 n - 1개가 작업을 계속할 수 있는 것

4. 고성능이 여러 독립적인 작업들에 대한 높은 처리량을 의미하는 경우도 존재하는데, 이를 태스크 수준(Task-Level Parallelism) 혹은 프로세스 수준 병렬성(Process-Level Parallelism)이라고 함
   - 태스크 수준 병렬성 또는 프로세스 수준 병렬성 : 다수의 프로세서를 이용해 독립적인 프로그램 여러 개를 동시에 수행하는 것
   - 이 때, 병렬로 수행되는 작업 하나 하나는 독립적 응용 프로그램으로, 이런 방식은 흔하면서도 중요한 병렬 컴퓨터 사용 방법 중 하나
     + 이는 여러 개 프로세서가 한 작업을 나누어서 수행하는 것과 대조되며, 이를 구별하기 위해 1개의 프로그램이 여러 프로세서에서 동시에 수행될 때 병렬 처리 프로그램(Parallel Processing Program)이라는 용어를 사용
     + 병렬 처리 프로그램 : 여러 개의 프로세서를 동시에 사용하는 프로그램

5. 클러스터(Cluster) : 근거리 네트워크(LAN)을 통해 연결된 컴퓨터들이 하나의 대형 멀티프로세서로 동작하는 시스템
   - 클러스터는 과학 분야 외에도 검색 엔진, 웹 서버, 전자우편 서버, 데이터베이스와 같이 부하가 많은 응용에도 사용될 수 있음

6. 멀티코어 마이크로프로세서(Mutli-Core Micro-Processor) : 한 칩 안에 다수 개의 프로세서(코어)를 집적시킨 마이크로프로세서로, 오늘날 데스크 톱과 거의 모든 프로세서들은 멀티코어
   - 멀티코어 칩에서 프로세서를 코어(Core)라고 부르며, 코어의 수는 하드웨어 기술 개선에 따라 증가됨
   - 멀티코어는 대부분 실제 주소 공간을 공유하는 공유 메모리 프로세서(Shared Memory Process, SMP / 단일 실제 주소 공간을 갖는 병렬 프로세서)

7. 직렬적(Serial), 병렬적(Parallel), 순차적(Sequential), 병행적(Concurrent)이라는 용어의 구별

   - 열은 소프트웨어를 나타내는데 소프트웨어는 근본적으로 순차적이든지 병행적
   - 행은 하드웨어를 나타내며 하드웨어는 직렬적이거나 병렬적
   - 예를 들어, 컴파일러를 만드는 프로그래머는 컴파일러를 파싱 / 코드 생성 / 최적화 등이 단계적으로 진행되는 순차적 프로그램으로 생각
   - 반면에 운영체제를 만드는 프로그래머는 운영체제를 컴퓨터에서 실행 중인 독립적 작업들의 입출력 이벤트를 처리하는 프로세스들로 구성된 병행적인 프로그램으로 생각
   - 하드웨어와 소프트웨어를 다른 축으로 표시한 이유는 병행 소프트웨어가 직렬 하드웨어에서 수행될 수 있고, 병렬 하드웨어에서 수행될 수 있음을 보여주기 위해서임
   - 앞으로는 순차적이거나 병행적이거나 병렬 하드웨어에서 수행되는 소프트웨어는 병렬 처리 프로그램 혹은 병렬 소프트웨어라고 부름
