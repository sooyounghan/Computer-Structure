-----
### 일관성을 유지하기 위한 기본 기법
-----
1. 캐시 일관성을 가진 멀티프로세서에서 캐시는 공유된 데이터의 이동(Migration)과 복제(Replication) 기능을 제공
   - 이동 : 데이터는 지역 캐시로 이동하여 그곳에서 투명하게 사용될 수 있음
     + 이동은 먼 곳에 위치한 공유 데이터를 접근할 때 발생하는 지연과 공유 메모리에 대한 대역폭 요구를 줄여줌
   - 복제 : 공유 데이터가 동시에 읽힐 때, 캐시는 지역 캐시 안에 복사본을 만듬
     + 복제는 공유 데이터 읽기에 대한 접근 지연과 경쟁을 감소시킴

2. 이동과 복제를 지원하는 것이 공유 데이터를 접근하는 성능에 매우 중요하므로, 많은 멀티프로세서들이 캐시 일관성을 유지하기 위한 하드웨어 프로토콜을 도입하고 있음
   - 멀티프로세서에서 일관성을 유지하기 위한 프로토콜은 캐시 일관성 유지 프로토콜(Cache Coherence Protocol)이라고 부름
   - 캐시 일관성 유지 프로토콜의 구현의 핵심은 데이터 블록의 모든 공유 상태를 추적하는 것
  
3. 가장 널리 사용되는 캐시 일관성 유지 프로토콜은 스누핑(Snooping)
   - 이 방법에서는 실제 메모리 블록 데이터의 복사본을 가지고 있는 캐시들이 협력하여 일관성을 유지하고, 중앙 집중식으로 블록의 공유 상태를 관리하지 않음
   - 캐시들은 모두 방송형 전송 매체(버스 또는 네트워크)를 통하여 접근할 수 있음
   - 그리고 모든 캐시 제어기들은 버스나 스위치 접근이 요청한 블록의 복사본을 가지고 있는지 없는지를 확인하기 위해 전송 매체를 감시(Snoop)함

4. 스누핑 프로토콜은 쉽게 구현할 수 있지만, 확장성을 제한
