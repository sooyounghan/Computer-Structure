### 디스크 메모리
-----

1. 하드 디스크는 원판 집합으로 구성되어 있고 원판은 분당 5400 ~ 15000번의 속도로 회전
2. 금속 원판의 양측 면은 카세트나 비디오테이프와 같이 자성체로 코팅되어 있으며, 하드 디스크 상 정보를 읽고 쓰기 위해서 읽기 / 쓰기 헤드라고 불리는 작은 전자기 코일을 갖고 있는 움직이는 암(Arm)이 각 표면 바로 위에 존재
3. 드라이브 내부를 보호하기 위해 전체 드라이브는 영구히 밀봉되어 있으며, 그 덕에 디스크 헤드가 드라이브 표면에 더 가까워질 수 있음
4. 각 디스크 표면은 트랙(Track)이라고 불리는 동심원으로 나누어짐
   - 일반적으로 표면당 수만 개의 트랙이 존재하고, 각 트랙은 다시 정보를 저장하는 섹터(Sector)로 나누어짐
     + 트랙(Track) : 자기 디스크의 표면을 이루고 있는 수십 만개의 동심원 중 하나
     + 섹터(Sector) : 자기 디스크의 한 트랙을 구성하는 여러 세그먼트 중 하나, 디스크에 정보를 쓰거나 읽는 가장 작은 단위
   - 한 트랙은 수천 개의 섹터로 구성되며, 섹터 크기는 보통 512바이트에서 4096바이트 정도임
   - 자성 매체에는 섹터 번호, 공백, 에러 정정 코드를 포함하는 섹터 데이터 / 공백 / 다음번 섹터의 번호와 같은 순서로 기록

5. 각 면의 디스크 헤드는 서로 연결되어 있으며 함께 움직이므로 모든 헤드는 각 면의 같은 트랙에 위치하게 됨
6. 헤드 아래에 있는 모든 면의 트랙을 실린더(Cylinder)라고 부름
7. 데이터에 접근하려면 운영체제가 3단계에 걸친 명령을 디스크에 내려야 함
   - 첫 단계는 적절한 트랙 위에 디스크 암을 갖다 놓는 일 : 이 작업은 탐색(Seek, 디스크 상 적합한 트랙에 읽기 / 쓰기 헤드를 위치시키는 동작)이라고 부르며, 디스크 헤드를 원하는 트랙까지 이동시키는데 걸리는 시간을 탐색 시간(Seek Time)이라고 함
   - 디스크 생산자는 제품 설명서에 최소 탐색시간, 최대 탐색시간, 평균 탐색시간을 표시
      + 처음 2개는 측정하기 쉽지만, 평균은 탐색 거리에 따라 달라지므로 여러 가지로 해석
      + 산업체에서는 평균 탐색 시간을 모든 탐색 시간의 합을 모든 탐색 횟수로 나눈 것으로 정의 : 일반적으로 평균 탐색시간이 3 ms ~ 13 ms로 광고
      + 그러나 디스크 접근에는 지역성이 있으므로 응용과 디스크 스케줄링에 따라 실제 평균 탐색 시간은 광고하는 값의 25% ~ 35% 정도임
      + 같은 파일을 연속적으로 접근하거나 운영체제가 디스크 요청을 묶어서 함께 스케줄하기 때문에 지역성이 생김

8. 헤드가 원하는 트랙에 도달하면 읽기 / 쓰기 헤드 밑에 원하는 섹터가 올 떄까지 기다려야 하는데, 이 시간을 회전 지연 시간(Rotational Latency 또는 Rotational Delay)라고 부름
   + 회전 지연 시간 : 원하는 섹터가 읽기 / 쓰기 헤드 밑에 도달할 때 까지의 시간으로, 일반적으로 회전 시간의 절반으로 가정
   + 원하는 정보에 도달하는 평균 회전 지연 시간은 디스크가 1/2 회전하는 데 걸리는 시간과 같음
   + 디스크이 회전 속도는 5400 ~ 15000 RPM이며, 5400 RPM일 때 평균 회전 지연 시간

9. 디스크 접근의 마지막 요소는 블록 하나를 전송하는 시간, 즉 전송시간(Transfer Time)
    - 전송시간은 섹터의 크기, 회전 속도, 트랙의 저장 밀도 함수
    - 2020년 기준 전송 속도는 150 MB/sec ~ 250 MB/sec 사이임

10. 대부분의 디스크 컨트롤러는 헤드 아래를 통과하는 섹터들을 저장하는 내장 캐시를 가지고 있으며, 캐시는 전송 속도가 더 빨라 2020년 기준으로 1500 MB/sec(12 Gbits/sec)까지 나옴
11. 기술이 발달하면서 블록을 배치하는 방법이 복잡해짐
    - 섹터-트랙-실린더 모델에서는 가까운 블록이 같은 트랙에 있는 것으로 가정 : 같은 실린더에 있는 블록은 탐색이 필요 없으므로 빨리 접근이 가능하며, 어떤 트랙들은 다른 트랙들보다 가까움
    - 블록 배치 방법이 달라진 것은 디스크 인터페이스 수준을 더 높였기 때문임
      + 순차적 전송을 더 빠르게 하기 위해 이 상위 수준 인터페이스는 디스크를 무작위 접근 장치보다는 테이프 같은 형태로 구성
      + 최고의 성능을 얻기 위해 같은 비트 밀도로 저장된 모든 섹터를 사용하려는 시도로 논리적 블록들을 디스크 표면에 구불구불한 순서로 배치
      + 따라서, 연속된 주소의 블록도 다른 트랙에 있을 수 있음

12. 자기 디스크와 반도체 메모리 기술 사이의 두 가지 큰 차이점은 디스크는 기계장치이므로 접근시간이 길고(플래시는 1000배 정도 빠르고, DRAM은 100,000배 정도 빠름), 디스크는 그리 높지 않은 가격에 아주 큰 저장 용량을 갖기 때문에 훨씬 쌈(디스크가 6배에서 300배 정도 쌈)
13. 자기 디스크는 플래시와 같이 비휘발성이지만, 플래시와 다르게 마모 문제가 없음
    - 그러나 플래시는 훨씬 튼튼해서 거친 환경에서 사용되는 개인 휴대용 기기에 더 잘 맞음
