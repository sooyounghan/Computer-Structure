### DRAM
-----
1. SRAM에서는 전력이 공급되는 한 그 값이 무한히 유지됨
2. DRAM에서는 셀이 기억되는 값이 전하 형태로 커패시터에 저장
   - 저장된 값을 읽거나 새로운 값을 쓰기 위해 저장된 전하를 접근하는 데 트랜지스터가 하나 필요

3. DRAM은 저장된 비트 하나당 트랜지스터 하나만 있으면 되므로 SRAM에 비해 훨씬 더 집적도가 높고 값도 쌈
   - DRAM은 커패시터에 전하를 저장하므로 무한히 유지할 수 없어 주기적으로 리프레시해야 함

4. 이 메모리 구조는 이렇게 기억을 지속하지 못하므로 SRAM 셀의 정적(Static) 저장과 반대로 동적(Dynamic)이라고 함
5. 셀의 리프레시는 단순히 셀에 저장된 값을 읽어서 다시 쓰면 됨
   - 전하는 수 밀리초 동안만 유지될 수 있음
   - 모든 비트를 하나씩 DRAM에서 읽어서 다시 쓴다면 계속 DRAM을 리프레시 해야하므로 기억된 값에 접근할 시간이 없게 됨
   - DRAM은 이를 위해 2단계 디코딩 구조를 갖기 때문에, 전체 행(워드라인을 공유하는)을 한꺼번에 읽는 읽기 사이클 후 바로 쓰기 사이클을 실행하여 한 행을 통쨰로 리프레시 가능

8. 리프레시에 도움이 되는 행 구조는 성능에도 도움이 됨
   - DRAM은 성능 향상을 위해 행을 버퍼링해서 반복적으로 접근
   - 버퍼는 SRAM처럼 동작하며, 다음 행에 접근하기 전까지 주소값만 바꾸면 버퍼 내 아무 비트나 접근이 가능
   - 이 기능은 같은 행 내의 비트 접근시간을 훨씬 주여주므로, 접근 시간을 크게 개선
   - 칩의 폭을 넓히면 칩의 메모리 대역폭 또한 커지게 됨
   - 버퍼에 있는 행은 DRAM의 폭(보통 4, 8, 또는 16비트)이 얼마든지 간에 연속된 주소를 사용해서 전송하거나 버퍼 내 시작 주소와 블록 전송을 이용해서 전송할 수 있음

9. 프로세서와의 인터페이스를 더욱 향상시키기 위해 DRAM에 클럭을 추가할 수 있는데, 이를 SDRAM(Synchronous DRAM이라고 부름
    - 장점 : 클럭을 사용하므로 메모리와 프로세서를 동기화하는 시간이 필요 없음
    - SDRAM의 속도가 빠른 것은 주소를 여러 번 지정하지 않아도 한꺼번에 여러 비트를 전송할 수 있기 때문이며, 주소를 일일이 지정하는 대신 클럭이 연속적인 비트들을 버스터 모드로 전송 : 이 메모리를 DDR(Double Data Reate) SDRAM이라고 부름
      + 이 이름은 클럭의 상승 에지에서도 데이터가 전송되고 하강 에지에서도 데이터가 전송되어 대역폭이 2배가 됨
      + 클럭 속도와 데이터 전송 폭으로부터 기대할 수 있는 대역폭의 2배를 얻을 수 있음
    - 이 기술의 최신 버전은 DDR4라고 불리며, DDR4-3200 DRAM은 초당 32억(3200 X $10^{6}$) 번의 전송이 가능한데, 이는 1600MHz 클럭을 사용함을 의미

10. 이와 같이 큰 대역폭을 유지하기 위해서는 DRAM의 내부를 잘 설계해야 함 : 단순하게 빠른 행 버퍼 하나를 사용하는 대신에, 각자 별도의 행 버퍼를 가지고 있는 여러 개의 뱅크를 동시에 읽고 쓸 수 있도록 DRAM 내부를 구성할 수 있음
    - 한 주소를 여러 뱅크에 보내서 모든 뱅크가 동시에 읽고 쓸 수 있게 함
      + 예) 뱅크가 4개라면, 한 번의 접근시간으로 4개의 뱅크를 돌아가면서 접근해서 4배의 대역폭을 제공
      + 이 순환 접근 방식을 주소 인터리빙(Address Interleaving)이라고 함
    - iPad와 같은 개인 휴대용 기기는 낱개 DRAM을 사용하지만, 서버용 메모리는 DIMM(Dual Inline Memory Module)이라는 작은 기판 형태로 판매
      + DIMM에는 보통 4 ~ 16개의 DRAM이 있으며, 서버 시스템용으로 8바이트 폭으로 구성되는 경우가 많음
      + DDR4-3200 SDRAM을 사용하는 DIMM은 초당 8 X 3200 = 25,600 MB을 전송할 수 있음
      + 이런 DIMM은 대역폭 크기에 따라 이름을 붙여 PC25600이라 함
      + DIMM에는 DRAM이 여러 개 있지만, 특정 전송에 사용되는 것은 일부이므로 DIMM 내 같은 주소를 공유하여 동시에 활성화되는 칩들을 지칭하는 용어가 필요
      + 행이나 뱅크처럼 DRAM 내부에서 사용하는 용어와 혼동을 피하기 위해 DIMM의 칩들의 부분집합을 메모리 랭크(Memory Rank)라고 부름
