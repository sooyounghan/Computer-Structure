-----
### 간단한 캐시
-----
1. 캐시의 핵심적인 특성
   - 직접 사상 캐시
   - 쓰기 할당을 이용하는 나중 쓰기
   - 블록 크기는 4워드 (16바이트 또는 128비트)
   - 캐시 크기는 16 KiB, 블록은 1024개
   - 32비트 주소
   - 캐시 블록마다 유효 비트와 갱신 비트를 가지고 있음

2. 캐시 주소의 각 필드 계싼
   - 캐시 인덱스는 10비트
   - 블록 변위는 4비트
   - 태그 크기는 32 - (10 + 4) = 18비트

3. 프로세서와 캐시 사이의 신호
   - 1비트의 읽기(Read) 또는 쓰기(Write) 신호
   - 캐시 동작이 있는지 없는지 표시하는 1비트의 유효(Vaild) 신호
   - 32비트 주소
   - 프로세서에서 캐시로 가는 32비트 데이터
   - 캐시에서 프로세서로 가는 32비트 데이터
   - 캐시 동작이 완료되었음을 표시하는 1비트의 준비(Ready) 신호

4. 데이터 필드가 128비트인 것을 제외하면, 메모리와 캐시 사이의 인터페이스는 프로세서와 캐시 사이의 인터페이스와 같은 필드를 가지고 있음
5. 오늘날의 마이크로프로세서는 일반적으로 더 넓은 메모리 폭을 사용
   - 그래서 프로세서에서는 32비트나 64비트 워드를 다루지만, DRAM 제어기는 128비트인 경우도 있음
   - 캐시 블록을 DRAM 폭에 맞추면 설계가 쉬워짐

6. 메모리와 캐시 사이의 신호
   - 1비트의 읽기 또는 쓰기 신호
   - 메모리 동작이 있는지 없는지를 표시하는 1비트의 유효 신호
   - 32비트 주소
   - 캐시에서 메모리로 가는 128비트 데이터
   - 메모리에서 캐시로 가는 128비트 데이터
   - 메모리 동작이 완료되었음을 표시하는 1비트의 준비 신호

7. 메모리 인터페이스는 고정된 수의 클럭을 사용하지 않음을 유의
   - 메모리 읽기 또는 쓰기가 끝났을 때 준비 신호를 통해서 캐시에게 알려 줄 메모리 컨트롤러가 있다고 가정
