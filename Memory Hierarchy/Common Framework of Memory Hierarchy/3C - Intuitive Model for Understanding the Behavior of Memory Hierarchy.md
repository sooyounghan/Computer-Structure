### 3C : 메모리 계층의 행동을 이해하기 위한 직관적 모델
-----
1. 3C 모델 (모든 캐시 실패가 필수 실패, 용량 실패, 대립 실패 세 가지 중 하나로 분류되는 캐시 모델)
   - 필수 실패(Compulsory Miss) : 캐시에 단 한번도 들어오지 않았던 블록에 대한 첫 번쨰 접근에 의해 발생하는 캐시 실패로, 이를 최초 시작 실패(Cold-Start Miss)라고도 함
   - 용량 실패(Capacity Miss) : 캐시가 프로그램 수행 중 필요한 모든 블록을 포함할 수 없을 때 발생하는 캐시 실패로, 완전 연관 캐시에서도 발생하며, 용량 실패는 블록이 교체되고 나중에 다시 그블록을 가져올 때 발생
   - 대립 실패(Conflict Miss) : 충돌 실패(Collision Miss)라고도 하며, 집합 연관이나 직접 사상 캐시에서 여러 블록들이 같은 집합에 대해 경쟁을 벌일 때 발생하는 캐시 실패로, 대립 실패는 같은 크기의 완전 연관 캐시에서는 발생하지 않음

2. 실패율이 어떻게 세 가지 요인으로 나누어지는지 보여주는 그림

   - 캐시 설계의 어떤 측면을 변경하면 이 실패의 원인들을 바로 줄일 수 있음
   - 대립 실패는 같은 캐시 블록에 대한 경쟁 때문에 발생하므로 연관 정도를 높이면 줄일 수 있음
   - 그러나 연관 정도의 증가는 접근시간을 느리게 해서 전체 성능을 떨어트릴 수 있음
   - 용량 실패는 캐시의 크기를 크게 해서 줄일 수 있음
     + 실제로 최근 수년 동안 2차 캐시 크기는 꾸준히 커지고 있음
     + 물론 캐시를 크게 하면 접근시간이 늘어나고 따라서 전체 성능을 저하시킬 수 있음을 주의 : 이런 이유로 1차 캐시의 크기는 느리게 성장

3. 필수 실패는 블록의 맨 처음 참조에 의해 발생되므로, 필수 실패의 수를 줄이기 위한 첫 번째 방법은 블록의 크기를 늘리는 것
   - 이렇게 하면 프로그램이 더 적은 수의 캐시 블록들로 이루어지기 때문에 프로그램의 모든 블록을 한 번씩 접근하는데 필요한 참조 횟수를 줄일 수 있음
   - 하지만, 블록 크기를 너무 크게하면 실패 손실이 늘어나므로 성능에 나쁜 영향을 줄 수 있음

4. 실패를 3C로 분해하는 것은 유용한 정성적 모델
   - 하지만 실제 캐시 설계에서는 많은 설계 대안들이 상호작용 하므로, 한 가지 캐시 특성을 바꾸면 실패율의 여러 요소에 영향을 끼치는 경우가 많음
   - 이러한 단점에도 불구하고 이 모델은 캐시 설계의 성능에 대한 통찰력을 주는 방법

5. 실패율을 개선할 수 있는 모든 변경들이 전체 성능에 악영향을 끼칠 수 있으므로 메모리 계층 설계를 어렵게 함
