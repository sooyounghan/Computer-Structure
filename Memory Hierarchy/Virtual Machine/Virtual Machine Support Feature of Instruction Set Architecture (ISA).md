-----
### 명령어 집합 구조의 가상 머신 지원 기능(의 결여)
-----
1. VM이 ISA를 설계하는 동안 계획되었다면, VMM이 실행해야 하는 명령어 개수를 줄이고 에뮬레이션 속도를 개선하기가 상대적으로 쉬웠을 것
   - VM이 하드웨어에서 직접 실행될 수 있는 구조에 가상화 기능(Virtualizable)을 부여

2. 비교적 최근에 와서야 PC와 서버 응용에 VM이 고려되기시작했으므로, 대부분 명령어 집합들은 가상화를 염두에 두고 만들어져 있음
3. VMM은 손님 시스템이 가상 자원하고만 상호작용하는 것을 보장해야 하므로, 보통의 손님 OS는 VMM 위에 사용자 모드 프로그램으로 동작
   - 만약 손님 OS가 특권 명령어를 이용해 하드웨어 자원과 연관된 정보를 접근하거나 조작하려는 시도를 하면(예를 들어, 인터럽트를 허용하는 상태 비트를 읽기나 쓰기) 트랩을 발생시켜 VMM을 부름
   - 그러면 VMM이 해당 실제 자원에 적절한 변경을 일으킬 수 있음

4. 따라서, 이처럼 민감한 정보를 읽거나 쓰려고 하는 명령어가 사용자 모드에서 실행될 때 트랩이 발생되게 하면, VMM이 이를 가로채 손님 OS가 기대하는 대로 이 민감한 정보의 가상 버전을 지원할 수 있음
5. 이런 지원 기능이 없으면 다른 조치를 취해야 함
   - VMM이 특별히 주의를 기울여 문제 소지 명령어를 모두 찾아내고, 이 명령어들이 손님 OS에 의해 실행될 때 올바르게 동작하도록 해야함
   - 이로 인해서, VMM의 복잡도가 증가하고 VM의 실행 성능이 떨어지게 됨
