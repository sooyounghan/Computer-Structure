-----
### 쓰기 처리
-----
1. 캐시의 접근시간과 메인 메모리 접근 시간 차이는 수십에서 수백 사이클 정도이며, 쓰기 지연 시간을 프로세서에게 안 보이게 하기 위해 쓰기 버퍼가 필요함
   - 물론, 즉시 쓰기 기법을 이용할 수 있음
     + 하지만, 가상 메모리의 계층 구조의 다음 계층(디스크)에 쓰는 데 수백만 사이클이 소요됨
     + 그러므로 쓰기 버퍼를 설치해서 디스크에 즉시 쓰기를 하는 것은 매우 비실용적
   - 그 대신에 가상 메모리 시스템은 나중 쓰기 방식을 사용
   - 쓰기 동작은 메인 메모리 내 페이지에만 수행하고, 페이지가 메모리에서 교체될 때 디스크에 복사

2. 가상 메모리 시스템에서 나중 쓰기 방식은 또 다른 큰 장점이 존재
   - 디스크 전송 시간이 디스크 접근 시간보다 짧기 때문에, 한 워드씩 디스크에 쓰는 것보다 페이지 전체를 한 꺼번에 쓰는 것이 훨씬 더 효율적
   - 나중 쓰기 방식이 각각의 워드를 전송하는 것보다 능률적이지만 대가가 존재
     + 교체할 페이지가 결정되면, 무조건 디스크에 쓰지 말고 이 페이지를 디스크에 쓸 필요가 있는지 없는지를 알아봐야 함
   - 메모리에 적재된 이후 페이지에 쓰기를 한 적이 있는지 표시하기 위해, 페이지 테이블에 갱신 비트(Dirty Bit)를 추가
     + 페이지 내 어떤 워드에 쓰기를 하면 이 갱신 비트를 1로 함
   - 운영체제가 내보낼 페이지를 선택하면, 새로운 페이지에게 공간을 내주기 전 그 페이지를 디스크에 써야 할지 말지를 갱신 비트가 알려줌
   - 그래서 수정된 페이지를 종종 갱신 페이지(Dirty Page)라고 부름
