-----
### 결론
-----
1. 메모리 접근이 유발하는 긴 시간지역을 극복할 기회를 주는 것이 지역성의 원칙
   - 여러 수준 메모리 계층구조가 정량적 측면에서 다르게 보일지 모르지만, 동작은 비슷한 전략을 따르며 똑같은 지역성 원칙을 활용하고 있음

2. 다단계 캐시는 두 가지 이유때문에 다른 최적화 기법을 더 쉽게 쓸 수 있게 해줌
   - 하위 계층 캐시의 설게 파라미터는 1차 캐시와는 다름 : 예를 들어, 하위 계층 캐시는 훨씬 크기 때문에 더 큰 블록을 사용할 수 있음
   - 하위 계층 캐시는 1차 캐시와 달리 프로세서가 계속해서 사용하지 않음
   - 따라서, 하위 계층 캐시가 쉬고 있을 때 앞으로 발생할 실패를 막기 위해 어떤 일을 하는 것을 고려할 수 있음

3. 다양한 프로그램 변형과 하드웨어 장치를 이용해 메모리 계층 구조를 효과적으로 관리하는 것이 컴파일러 분야의 주요 연구 쟁점
   - 공간적 및 시간적 지역성을 높이기 위해 프로그램을 재구성하는 것
   - 선인출(Prefetching) : 장래에 필요할 데이터 블록을 블록 주소를 가리키는 특수 명령어를 사용해, 데이터 블록이 실제로 참조되기 전에 캐시로 미리 가져오는 것
     + 많은 마이크로프로세서가 하드웨어 선인출을 사용해 소프트웨어가 알아채기 어려운 접근의 예측을 시도
   - 메모리 전송을 최적화하는 특수한 캐시 인지(Cache-Aware) 명령어를 사용하는 것
